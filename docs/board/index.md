# Board

The **Board** view is the main Kanban workspace. Each board has **columns** (workflow stages) and optional **swimlanes** (grouping). Task **cards** live in cells; you open a card to edit its title, description, links, and other fields.

## What you can do

- **Switch boards** from the board name control in the header (when the repo has more than one board).
- **Filter by owner** so you only see your work or a specific person’s cards.
- **Search** to jump to open or closed cards that match.
- **Move cards** by dragging and dropping, or use the **card compass** (move buttons) on a selected card.
- **Sync** to run `git pull` and `git push` for the repo; the control reminds you when there are local changes to share.
- A short **completed** strip at the bottom shows a few recently closed cards; use the [Completed](../completed/) view for the full list and paging.

Card descriptions support limited **Markdown** (see [supported markdown](../markdown.md)).
